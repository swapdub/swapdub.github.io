Windows Scripting

A short write-up on automating Windows with batch scripting — minimal installation, practical result.

I recently got the urge to automate a genuinely annoying part of my gaming setup. The idea had been floating around for a while, nudged along by a recurring Udemy course that pops up free every other week. I finally caved.

The Problem

League of Legends divides its player base by region — a great design choice for latency, a terrible one if you have friends on both sides of the planet. Playing on both NA and EUW means two accounts. Their client famously cannot remember multiple logins, so every session starts with typing credentials manually. Riot Games' login UX in 2021: ☆☆☆☆☆.

The Objective

Write a script for each region that: opens the League client, waits for it to load, then automatically types the correct username and password into the login fields. One double-click — done.

Why Batch?

Of the realistic options — Python (with PyAutoGUI), AutoHotkey, or plain Batch — I went with Batch scripting. Zero installations required. It ships with every version of Windows and, for a task this focused, the overhead of spinning up a Python environment wasn't worth it.

The core trick is using timeout to wait for the client splash screen to clear, then mshta vbscript for sending keystrokes, since Batch doesn't have native input simulation.

Tools
